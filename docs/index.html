Hard-won lessons

Rules baked in

Things that would take hours to debug without prior experience.

No shell injection

${{ }} expressions never go in run: steps — always moved to env: blocks.

npm ci --include=optional

Plain npm ci drops optional deps on Linux runners. --include=optional preserves lockfile determinism and fixes cross-platform native bindings.

Deploy via workflow_run, never on push

Deploy fires only after CI concludes with success — never triggered directly by a tag push.

Tag deploys are never cancelled

cancel-in-progress is conditional — only PR previews can be cancelled, never environment deploys.

Least-privilege permissions

Every workflow declares a permissions: block. Without it, GITHUB_TOKEN inherits repo-wide write access.

workflow_run name must match exactly

The workflows: [CI] string must match the name: field in ci.yml exactly — a mismatch silently never triggers the deploy.
